Tenant Model

One tenant owns one or more agents. In most products, one real-world entity maps to one agent: one horse, one receptionist, one assistant persona.

Trusted Control Plane

Runtime API

Resolves tenant + agent, loads policy, gathers context, and routes the request.

Policy Service

Hard capabilities, allowed skills, device access, and product boundaries.

Memory Service

Maintains bounded prompt memory and runs Hermes-style post-turn memory updates.

Knowledge Service

Curated domain knowledge that stays separate from conversational memory.

Gateway Service

The only privileged path for sandboxed execution to reach models, storage, and system state.

Skill Registry

Stores sealed skill manifests, versions, runtime classes, and agent-level installations.

Agent Memory Shape

MEMORY.md Small always-on distilled memory for facts, routines, lessons, and stable context.
AGENT_PROFILE.md Profile summary for identity, role, preferences, and long-lived behavioral framing.
SQLite Session History Long-term recall archive searched on demand instead of injected every turn.

Audit and Observability

Captures policy decisions, memory changes, skill execution, device events, and operator-visible traces.

Execution Plane

Self-Hosted Kubernetes on Linux

Runs the platform services and isolated execution workloads on your own infrastructure.

Core Runtime

Cheap path for memory-first turns. Reads prompt memory, searches session history, and answers without sandboxing when no skill is needed.

Memory-First Path

gVisor Skill Runner

Default sandbox tier for higher-risk skill execution. Connected to the platform only through the gateway service.

Default Sandbox Tier

Future High-Isolation Tier

Kata or micro-VM runtime for premium customers or hostile workloads that need stronger boundaries.
